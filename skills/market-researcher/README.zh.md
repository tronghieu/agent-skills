# 市场调查员 (Market Researcher)

**Language / Ngôn ngữ / 语言:** [English](./README.md) | [Tiếng Việt](./README.vi.md) | [中文](./README.zh.md)

此 Skill 将您的 AI 助手转化为一位严谨自律的**桌面市场调查分析师 (desk research analyst)**——一位其笔下每一个数字您都能点击追溯到来源的分析师，无论是五分钟的 go/no-go 快速扫描，还是横跨多次会话的深度市场调研。

## Market Researcher 是什么？

让一个普通聊天机器人帮您测算市场规模，您得到的只是一个自信满满、却无从核实的数字——一个凭空捏造的 42 亿美元 TAM，看起来和一个真实的数字毫无区别。这个 Skill 的存在，就是为了让这种失败可以被发现：它接收一个原始问题，例如"X 有市场吗？"、"巴西的 Y 市场有多大？"、"我们应不应该在印尼推出这款产品？"，然后返回一份报告，其中**每一条事实性论断都能追溯到一个已登记的来源**。它适用于任何市场、任何国家、任何语言——肯尼亚一个 5000 万美元的细分市场，和美国一个 500 亿美元的大品类，都会得到同等的严谨对待。

此 Skill 只运行**桌面研究 (desk research)**：网络来源、公开数据、您提供的文档。它对这个边界很诚实——从中了解到的客户信息只是需求*信号*和*假设*，绝不是已验证的洞察（见下文不可动摇原则第 5 条）。当您需要真正的客户验证时，此 Skill 会把开放问题移交给一手研究（primary research），而不是假装替您回答。

## 为什么使用它？

- **拒绝没有来源的数字。** 每个数字都带有来源标签 `[S#]`，指向 `sources.md` 中的一行记录；或者是一个明确展示公式与引用输入的推导估算；或者被标注为 `(assumption A#)`（假设）。没有标签的事实性陈述会被视为一个错误。
- **事实来自网络，而非记忆。** 市场规模、价格、竞争对手动向、监管政策——每次都重新研究。训练数据既陈旧，又更糟糕地"看起来统计上很合理"；如果无法访问网络，Skill 会明确说明，并停止工作，而不是退化成一篇没有引用的文章。
- **交叉验证 (triangulate)，绝不悄悄取平均。** 当两个可信来源或方法的结果相差超过约 2 倍时，报告会揭示这一矛盾并诊断原因（市场定义不同、年份不同、货币不同、收入与 GMV 的差异），而不是把它们混合抹平。
- **交付前必经核实。** 一次"怀疑者"审查会将报告中最影响决策的论断与其来源逐一核对，重新计算每一项推导估算，并标记过时或单一来源的数据——所有这些都发生在您看到报告之前。
- **桌面研究只产出信号，而非已验证的洞察。** 这里产出的用户画像和需求都是有待一手研究验证的假设，始终明确标注，并始终以"留给一手研究的开放问题"收尾。

## 五条不可动摇的原则

1. **每一条论断都可追溯**——一个来源标签 `[S#]`、一个展示公式的推导估算，或一个明确的 `(assumption)`（假设）。
2. **最新事实来自网络，而非记忆**——陈旧的训练数据不能冒充研究。
3. **交叉验证；绝不悄悄取平均**——来源之间的分歧会被揭示并诊断，而不是被抹平。
4. **交付前必须运行一次核实审查**——由一位"怀疑者"复核关键论断和计算过程。
5. **桌面研究产出信号，而非已验证的客户洞察**——用户画像和需求都是假设，始终标注清楚，始终交由一手研究去确认。

## 两种模式

| | **Quick Scan（快速扫描，默认）** | **Deep Dive（深度调研，可选）** |
|---|---|---|
| 核心问题 | "这值得进一步研究吗？" | "我们究竟要面对什么？" |
| 时长 | 一次会话 | 多次会话，可随时续接 |
| 规模测算 | 一种方法 + 合理性检查，给出诚实的宽区间 | ≥2 种方法，交叉验证 |
| 竞争对手 | 识别 5–10 个，一句话概要 | 深度剖析前 3–5 名 |
| 客户 | 头部需求信号 | 完整信号挖掘 + 假设性用户画像 |
| 宏观环境 | 仅 2–3 个致命因素 / 利好因素 | 结构化 PESTEL + 趋势分析 |
| 输出 | Go/no-go 简报（约 4–8 页） | 模块化报告 |

Agent 默认运行 Quick Scan，并会提前明确说明这一点。Quick Scan 遵循一个硬性工作预算——**每个 lane 约 15 个已登记来源**——足以框定答案范围，而非彻底敲定答案。Deep Dive 是模块化的：您可以只选择重要的 lane（规模测算、竞争对手、需求、宏观），而不必机械地把四个都跑一遍；它也可以跨会话续接——在新会话中说一句 `status`，Agent 就会读取 `state.md`，从上次停下的地方精确接续。

## 四个研究 Lane

| Lane | 产出内容 |
|---|---|
| **市场与规模测算** | 通过自下而上和/或自上而下的方法计算 TAM/SAM/SOM，交叉验证后的区间与具名假设绑定 |
| **竞争对手分析** | 直接、间接、替代型以及"维持现状"竞争对手，分层并逐一剖析，给出定位判断 |
| **需求信号** | Jobs-to-be-Done 框架、痛点挖掘、付费意愿信号，诚实标注的假设性用户画像 |
| **趋势与宏观环境（PESTEL）** | 环境中哪些因素会成就或扼杀这个具体机会——绝不是一堂地理常识课 |

每个 lane 都可以作为并行子代理运行，拥有各自的来源预算、各自的发现文件，以及一段预留的 `[S#]` 编号区间，确保并行工作互不冲突。

## 组合契约 (Composition Contract)：可被其他 Skill 调用

Market Researcher 的设计初衷之一，就是被其他 Skill 和工作流当作研究引擎来调用——无论是正在构建事实基础的战略顾问委员会 (strategy board)，正在验证问题空间的 design-thinking 工作区，还是任何需要外部市场事实的产品简报流程。契约很简单：

- **输入：** 调用方提供研究问题与决策背景、市场定义（或请此 Skill 代为起草）、模式与所需 lane，以及其自身工作区内的目标目录。
- **输出：** 此 Skill 将结果写入该目录——追加到已有的 `sources.md` 中，并**延续其 `[S#]` 编号，绝不重新编号**已有内容——同时生成 `findings-*.md` 和 `report.md`，全部遵循同一套引用规范，调用方无需了解研究过程即可直接引用 `[S#]`。
- **边界：** 此 Skill 通过桌面研究回答市场问题。当请求超出市场研究范畴，或需要与真实客户进行一手研究时，它会明确说明，而不是勉强凑数。

## 它产出的工作区结构

```text
research/market/<topic-slug>/
  sources.md          # 来源登记表：ID、URL、发布方、日期、置信度
  findings-sizing.md         # 每个已运行的 lane 对应一份发现文件
  findings-competitors.md
  findings-demand.md
  findings-macro.md
  report.md            # 最终交付物：结论、关键事实、注意事项、开放问题
  state.md              # 模式、已完成的 lane、核实状态——用于后续续接
```

`scripts/init-research.sh` 负责搭建这个工作区（幂等操作——绝不覆盖已存在的文件，因此在调用方已有 `sources.md` 的工作区上重复运行也是安全的）。

## 如何触发

让 AI 执行如下任务：

```text
德国的独居人士订阅式 meal-kit 服务，这个市场存在吗？
```

```text
东南亚的 B2B 费用管理软件 TAM 有多大？
```

```text
加拿大中端会计 SaaS 的竞争对手有哪些，市场空白在哪里？
```

```text
Nghiên cứu thị trường xe điện hai bánh tại Việt Nam — có đáng để làm không?
```

```text
市場調査をお願いします。日本の高齢者向け配食サービス市場は参入する価値がありますか？
```

无论使用哪种语言、面向哪个国家的市场，只要请求涉及研究、测算规模、验证或进入某个市场，此 Skill 都会触发——不需要特意说出"市场调查"这几个字。

## 安装

### 1. 使用 CLI（推荐）

```bash
npx skills add tronghieu/agent-skills --skill market-researcher
```

### 2. 手动安装（适合非技术用户）

1. **下载：** 转到此仓库的 `skills/` 文件夹下载 `market-researcher.zip` 文件。
2. **解压和复制：** 解压 `market-researcher.zip` 并将 `market-researcher` 文件夹复制到以下目录之一：

**针对特定项目：**
将 `market-researcher` 文件夹复制到项目根目录下的 `.agents/skills/` 或 `.claude/skills/`。

**全局安装（所有项目可用）：**
* **Mac / Linux：** `~/.agents/skills/` 或 `~/.claude/skills/`
* **Windows：** `%USERPROFILE%\.agents\skills\` 或 `%USERPROFILE%\.claude\skills\`（通常为 `C:\Users\<YourUsername>`）

## 使用示例

> **您：** 我们正考虑在墨西哥推出一款 B2B 开票 SaaS，这个市场值得进入吗？
>
> **Agent：** 我先运行一次 Quick Scan——在一次会话内做一次全面、诚实的扫描；之后我们可以针对任何看起来关键的 lane 做深度调研。先问一下：您瞄准的是小微企业、中端市场，还是两者都要？这次只做墨西哥本地，还是把它当作拉美市场的立足点？
>
> *（确认问题框架，初始化工作区，运行规模测算 / 竞争对手 / 需求 / 宏观四个 lane——每个 lane 都阅读新鲜的网络来源，为每条论断打上标签，并以 `[S#]` 编号登记来源）*
>
> **Agent：** 结论：**值得进入，中等置信度。** 墨西哥的正规中小企业细分市场规模可观，且本土玩家覆盖不足 [S4][S11]；但付费意愿信号最强的群体，恰恰是那些*已经*在使用某种开票软件的企业 [S9]——而您真正想争取的非正规部门多数群体，相关信号要薄弱得多。核实阶段发现的最大注意事项：三个市场规模数字中有两个都可追溯到同一家供应商的报告系列 [S3][S7]，因此区间上限需谨慎对待。完整报告、来源清单和留给一手研究的开放问题都在 `research/market/mx-invoicing-saas/report.md` 中。
