# 战略顾问委员会 (Strategy Board)

**Language / 语言:** [English](./README.md) | [Tiếng Việt](./README.vi.md) | [中文](./README.zh.md)

此 Skill 将您的 AI 助手转化为一个虚拟的 **C 级战略顾问委员会**——由以战略学界巨匠命名的专家团队组成，带您从一个原始的战略问题走到一份经得起答辩、可直接提交董事会的建议方案。

## Strategy Board 是什么？

您不再是让一个普通聊天机器人"帮忙想想战略"，而是召集一整个委员会。**Drucker**（德鲁克）担任管理合伙人主持全局——界定真正的问题、指派合适的专家、并贯穿整个项目的主线——同时**七位专家**各自把守一个视角：市场、创新、财务、执行、情景规划、风险与综合表达。每个角色都刻意收窄关注范围，以确保单项评估被认真做透：一位同时要"推销"建议的分析师，会不自觉地把数字做软；一位同时兼任风险官的乐观主义者，永远发现不了致命缺陷。让不同的视角有组织地碰撞，正是委员会优于单一顾问之处。

**决策权始终在您手中。** 委员会负责研究、分析、辩论并提出建议——但从不代替您做决定。每一次项目都以把清晰的选择权交还给真正承担结果的人而结束。

## 为什么使用它？

- **拒绝凭空编造数字。** 输出中的每一个数字都必须标注来源，或明确标记为 `(assumption)`（假设）。自信地编造数字是 AI 战略顾问最糟糕的失败模式——这个 Skill 从设计上就拒绝走这条捷径。
- **先有事实，后做分析。** 委员会会先建立有据可查的事实基础，再运行任何分析框架，避免框架被套用在猜测之上。
- **始终提供三个真实的方案。** 绝不会用一个建议伪装成"多个选择"——总会有三个方向，每一个都值得一位理性的董事会成员为其辩护，如果诚实的话，其中也可能包括"什么都不做"。
- **上会前必经红队审查。** 每一份建议在到达您手中之前，都要经过 Taleb 的审查或一次事前验尸（pre-mortem）。
- **战略即取舍。** 每一份建议都必须说明组织将*停止做什么*——一份没有任何取舍的战略，只是一张愿望清单。
- **Boardroom（圆桌会议模式）。** 面对存在真实分歧的决策，可召集 3–4 位成员就一个尖锐的问题展开辩论——先各自独立表态，再交叉讨论——并始终留下会议记录。

## 委员会成员

**Drucker**（德鲁克）——*彼得·德鲁克。* 管理合伙人：界定真正的问题、指派专家、把守决策日志。德鲁克本人从不亲自做专业分析。

| 成员 | 致敬对象 | 视角 |
|--------|---------|------|
| **Porter**（波特） | 迈克尔·波特 | 市场与竞争——产业结构、定位 |
| **Christensen**（克里斯坦森） | 克莱顿·克里斯坦森 | 创新与颠覆——待完成的工作（jobs-to-be-done）、三层地平线 |
| **Graham**（格雷厄姆） | 本杰明·格雷厄姆 | 财务与价值——规模测算、单位经济效益、敏感性分析 |
| **Grove**（格鲁夫） | 安迪·格鲁夫 | 执行与组织——能力、运营模式 |
| **Wack**（瓦克） | 皮埃尔·瓦克 | 情景与不确定性——未来图景、"必须成立的前提" |
| **Taleb**（塔勒布） | 纳西姆·塔勒布 | 红队与风险——事前验尸、脆弱假设 |
| **Minto**（明托） | 芭芭拉·明托 | 综合与沟通——金字塔原理、董事会汇报材料 |

## 项目推进流程 (Engagement Pipeline)

针对一个真实待决的决策，委员会会按照一套设有"关卡"的流程推进——每个阶段都产出一份具体的成果，其中三个关卡会停下来，等待您明确点头才能继续：

1. **Brief（简报）** ——Drucker 把您的请求转化为一个已确认的决策问题。⛔ *关卡：您确认它。*
2. **Fact base（事实基础）** ——委员会研究事实，每个数字都有据可查。
3. **Analysis（分析）** ——相关专家各自从自己的视角审视这项决策。
4. **Options（方案）** ——三个真实的方向，诚实地加以比较。⛔ *关卡：您选定一个。*
5. **Stress-test（压力测试）** ——Taleb 在现实动手之前，先尝试把选定的方向"打破"。
6. **Recommendation（建议）** ——Minto 把它写成文字，先给答案。⛔ *关卡：您批准。*
7. **Roadmap（路线图）** ——Grove 把已批准的选择转化为负责人、时间节点，以及为此需要停止做的事。

并非每个项目都需要把每个阶段做到最深——Drucker 会提出一个合适规模的计划，由您来做取舍。但以下几点始终不可省略：溯源纪律、三个方案、事前验尸，以及各个关卡。

对于单次分析型请求（例如"测算这个市场的规模"、"给这份计划做一次事前验尸"），委员会会跳过完整流程，直接指派对应的专家处理——溯源和严谨性的要求依然适用。

## 如何进行一次战略讨论

您是决策者，委员会为您服务。一个完整的项目要经历七个阶段，但您在每个阶段要做的事都很小、很具体：

1. **提出要决策的问题。** 用简单的话说出来——"我们要不要进入 X？"、"Y 是自建还是外购？"。不需要任何准备；Drucker 会开启这个项目。
2. **回答界定范围的问题（Phase 0）。** Drucker 会追问真正的问题、截止时间、范围、限制条件，以及您的风险偏好，然后诊断战略环境（Strategy Palette），并提出一个规模合适的计划。**关卡一——您确认问题本身**，之后才会投入任何分析。
3. **提供只有您才知道的信息（Phase 1）。** 委员会负责研究带引用来源的市场数据；而内部数字、团队产能和内部政治现实，必须由您提供。您无法提供的内容会成为一条具名的假设——而绝不是一个悄悄的猜测。
4. **阅读"那又如何"的部分（Phase 2）。** 各位专家运行各自的视角，各自给出自己的分析。在每个停顿点，您都会得到一份菜单：细化 / 深入 / 质疑它（交给 Taleb）/ 继续。
5. **选定方向（Phase 3）。** 委员会会在同一张诚实的表格中列出三个真实的方案。如果团队意见分歧，就开一场 Boardroom 会议，看成员们把问题辩清楚。**关卡二——选择权在您**，并与被否决的方案一同记录在案。
6. **让它接受攻击（Phase 4）。** Taleb 的事前验尸会假设您选定的方向已经失败，并解释原因。由您决定哪些风险需要缓解，哪些可以有意识地接受。
7. **批准并执行（Phase 5–6）。** Minto 会把一切浓缩成一份"先给答案"的建议——**关卡三——您批准它**——随后 Grove 将其转化为一份路线图（roadmap），列明负责人、时间节点，以及为此需要停止做的事。

更短的路径：如果只是单次分析请求（例如"测算这个市场的规模"），直接问即可——对应的专家会以一份独立完整的备忘录形式交付。要在新的会话中恢复一个已暂停的项目，输入 `status`——委员会会读取其记录，从您上次停下的地方继续。

## Boardroom（圆桌会议模式）

标准流程一次只处理一位专家——适合生产型工作。**Boardroom** 则用于存在真实分歧的决策场景：Drucker 召集 3–4 位相关成员，围绕一个尖锐的问题落座，先收集每位成员*独立*的第一反应（以避免群体思维），随后展开交叉讨论，由您来主导会场走向。每场会议都以会议记录和一项决定收尾。这一模式通常会在方案辩论阶段（Phase 3）自然被提出，并且——以其对抗性的事前验尸变体形式——在任何建议正式提交之前（Phase 4）必定运行一次。

## 五条不可动摇的原则

1. **不编造数字**——每个数字都必须有来源，或标注为 `(assumption)`。
2. **先事实，后分析**——事实基础必须先于任何分析框架建立。
3. **提出建议前必须有三个真实方案**——而不是一个真方案配两个陪衬。
4. **每份建议都必须经过红队审查**才能提交。
5. **战略即取舍**——每份建议都必须说明组织将停止做什么。

此外，每一部分分析都要经受一个恒定的检验："那又如何？"（So what?）——如果某一部分不会改变决策，就应该被删掉。

## 如何触发

让 AI 执行如下任务：

```text
我们应该自建仓储管理系统还是直接采购？请召集战略委员会帮我分析。
```

```text
请 Porter 出面，帮我用五力模型分析一下越南网约车市场。
```

```text
关于进入柬埔寨市场，我们现在有三个方案，团队意见分歧很大。开一场 Boardroom 会议吧。
```

## 一切都会被记录下来

委员会在工作过程中会把一切都写下来——简报、带来源的事实、每位专家的分析、方案、会议记录、建议、路线图，以及一份持续更新的决策日志，记录每一次决定及其理由。这一切都保存在您工作区里的同一个文件夹中，因此您可以随时结束对话，之后再从原来的地方接着做——在之后的会话中，只需询问一次 `status` 即可。

## 可选的市场研究辅助 Skill

当战略项目需要更深入的市场规模测算、竞争者梳理、需求信号或宏观趋势研究时，Strategy Board 会建议您选择性使用 [Market Researcher Skill](https://github.com/tronghieu/agent-skills#market-researcher)。它可以建立可追溯的外部事实基础，供委员会直接使用。

选择权始终在您手中：这是一项建议，而非必备条件。您可以安装并使用它，也可以继续使用 Strategy Board 自带的研究流程。

```bash
npx skills add tronghieu/agent-skills --skill market-researcher
```

## 安装

### 1. 使用 CLI（推荐）

```bash
npx skills add tronghieu/agent-skills --skill strategy-board
```

### 2. 手动安装（适合非技术用户）

1. **下载：** 转到此仓库的 `skills/` 文件夹下载 `strategy-board.zip` 文件。
2. **解压和复制：** 解压 `strategy-board.zip` 并将 `strategy-board` 文件夹复制到以下目录之一：

**针对特定项目：**
将 `strategy-board` 文件夹复制到项目根目录下的 `.agents/skills/` 或 `.claude/skills/`。

**全局安装（所有项目可用）：**
* **Mac / Linux：** `~/.agents/skills/` 或 `~/.claude/skills/`
* **Windows：** `%USERPROFILE%\.agents\skills\` 或 `%USERPROFILE%\.claude\skills\`（通常为 `C:\Users\<YourUsername>`）

## 使用示例

> **您：** 我们正在考虑明年把 SaaS 产品拓展到欧盟市场，委员会能帮忙看看吗？
>
> **Drucker：** 在指派任何人之前，我想先确认一下——真正的问题是"要不要拓展到欧盟"，还是"国内增长正在放缓，欧盟看起来是最快的杠杆"？这一点值得先厘清，因为它会决定我们该研究什么。[提出几个简短的编号问题以界定范围，然后起草简报]
>
> *（关卡一——您确认问题框架。）*
>
> **Drucker：** 我会指派 Porter 分析市场结构，Graham 做基础规模测算——这两项可以并行进行。[各专家基于事实基础展开工作，各自产出一份有据可查的分析]
>
> **Drucker：** 目前摆在桌面上的有三个方案：直接进入、通过分销伙伴合作，或先花 12 个月在英国建立立足点再决定。鉴于 Porter 的判断和 Graham 的数字方向不太一致，您要不要在做选择之前先开一场 Boardroom 会议？
>
> *（Boardroom 召集 Porter、Graham、Grove 和 Taleb；先各自独立表态，再交叉讨论；您主导会场走向；会议记录已保存。）*
>
> *（关卡二——您选定方向。）* → Taleb 运行事前验尸 → Minto 起草建议 → *（关卡三——您批准）* → Grove 将其转化为路线图。
