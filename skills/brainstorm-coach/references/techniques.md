# Technique library — router

Twenty classic brainstorming techniques, grouped into five files by
what they're good at. Every technique runs in the same hybrid rhythm —
one prompt, user answers first, AI builds, floor returns — so each group
file covers what makes its techniques distinct: the opening move, the
round shape, a worked example, how it commonly goes wrong, and when to
bail. This file is the router: use it to pick a technique or a group,
then open the file that has it.

## Selection guide

Match the shape of the session to the shape of the technique, then open
the file that technique lives in.

| Session goal / topic shape | Recommended techniques | Group file |
|---|---|---|
| Stale topic, need fresh angles | Random stimulation, Forced relationships | collaborative.md, advanced.md |
| Challenge the frame itself | Assumption reversal, Question storming | advanced.md |
| Improve an existing product or process | SCAMPER, Reversal / Inversion | structured-frameworks.md, creative-expansion.md |
| Explore a wide-open space, nothing fixed yet | What-if scenarios, Mind mapping | creative-expansion.md, structured-frameworks.md |
| See the topic through others' eyes | Role playing, Six Thinking Hats | advanced.md, structured-frameworks.md |
| Constraints feel suffocating or absent | Resource constraints, Time shifting | advanced.md |
| Team energy is flat, need momentum | Yes-and building, Brainwriting round-robin | collaborative.md |
| Want the deeper need behind a stated want | Five Whys (exploratory) | deep-exploration.md |
| Topic resists literal thinking | Metaphor mapping, Analogical thinking | advanced.md, creative-expansion.md |
| Genuine deadlock, same ideas on repeat | Provocation (PO), First-principles thinking | deep-exploration.md, creative-expansion.md |

**Default progressive flow.** For a full arc rather than one technique,
run broad → structured → narrowing:

1. **What-if scenarios** (creative-expansion.md) — open the space, no
   constraints yet.
2. **Analogical thinking** (creative-expansion.md) — pull in outside
   patterns before narrowing.
3. **SCAMPER** (structured-frameworks.md) — apply structured pressure to
   what's surfaced.
4. **Question storming** (advanced.md) — surface what's unresolved, as
   questions.
5. **Converge** — cluster and pick top 3 (session flow step 4–5).

Two to four techniques cover most sessions; this flow is a default to
offer, not a checklist to complete in full.

## Switching and sequencing

A technique is spent when ideas start repeating with different wording,
answers get shorter round over round, or the user drifts toward a
different topic unprompted — that's the signal to switch, not to push
harder. Pushing past its energy produces diminishing, samey ideas and
teaches the user that brainstorming is a chore.

Two to four techniques is plenty for a single session. Cycling through
six or seven rarely buys more ideas — it pays setup cost on each new
technique without staying anywhere long enough to go deep. When in
doubt, favor depth on fewer techniques over breadth across many. Each
group file's per-technique "where it goes wrong" and "bail when" notes
give the specific signal for that technique; this is the general
version for the session as a whole.
