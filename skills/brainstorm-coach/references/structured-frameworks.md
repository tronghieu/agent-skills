# Group 2 — Structured frameworks

Three techniques that impose an external structure — named lenses,
named hats, a branching map — on the topic, trading some open-endedness
for coverage a free-form session would drift past. Reach for this group
once there's something concrete to work on (an existing product,
process, or decision) rather than at the very start of a wide-open
session. The examples below use one running generic topic — a community
tool-lending library (a place where neighbors borrow tools instead of
buying them) — so the same idea can be seen through several lenses.

## 5. SCAMPER

**What it is** — Seven lenses applied one at a time to the topic:
Substitute, Combine, Adapt, Modify, Put to other use, Eliminate,
Reverse. Working through named lenses in sequence forces coverage a
free-form session would drift past — each letter points attention
somewhere the previous one didn't.

**Best for** — Improving an existing, concrete product, feature, or
process. SCAMPER needs something with actual parts to operate on; it
struggles on a topic that's still just an idea.

**Facilitate it** — One letter per round: "Substitute — what's one part
of the tool library we could swap out for something else entirely?"
Move to the next letter once the round's energy drops, not on a fixed
timer. Skip a letter outright if it plainly doesn't apply to the topic —
a strained round produces weaker ideas than the time spent would justify.

**Example round**
- Prompt: "Substitute — what's one part of the tool library we could
  swap out entirely?"
- *(user)*: "Swap the fixed building for a trailer that visits different
  neighborhoods on a schedule."
- *(AI) builds*:
  - *(AI) Substitute the checkout process too*: swap the front-desk
    clerk for a self-serve kiosk, freeing a person to actually teach
    tool safety instead of scanning barcodes.
  - *(AI) Substitute who supplies the tools*: swap "the library owns
    everything" for a consignment model where members lend their own
    idle tools for a cut of the fee.
  - *(AI) Substitute the payment model*: swap a flat membership fee for
    pay-per-borrow, lowering the barrier for occasional users.
- Return the floor: "Want to push on the trailer idea, or move to the
  next letter — Combine?"

**Your build move** — Apply the current letter's lens to a different
part of the topic than the user picked — SCAMPER's whole value is
coverage, so your build should extend the lens's reach, not add a
second idea to the same spot the user already covered.

**Where it goes wrong** — Grinding through every letter after the
energy clearly left three letters ago is the classic failure — forcing
an "Eliminate" round on a topic that has nothing left to eliminate
produces thin, obligatory answers. Watch for one-line, low-effort
replies as the signal, not the calendar of letters remaining.

**Bail when** — Two consecutive letters produce forced or repetitive
answers; stop cycling letters and either converge on what's already
surfaced or switch to a technique with a different shape, like Mind
mapping or Reversal.

## 6. Six Thinking Hats

**What it is** — Six perspectives, one per round, kept isolated from
each other so they don't collapse into a single muddled take: facts,
emotions, risks, benefits, creative possibility, and process. Separating
them stops "here's a wild idea" and "yes, but is it risky" from
happening in the same breath.

**Best for** — Genuinely different lenses on a topic, especially when
the default is to jump straight to risk or straight to enthusiasm and
skip the rest entirely.

**Facilitate it** — Name the hat for the round and stay strictly inside
it: "White hat — just the facts. What do we actually know about the
tool library idea, no opinions yet?" Then in sequence: red (gut
reactions), black (risks and weaknesses), yellow (benefits and
optimism), green (pure creative options), blue (process — how should we
be running this session?). One hat, one round; don't let a black-hat
worry sneak into the green-hat round.

**Example round**
- Prompt: "Black hat — risks and weaknesses. What could genuinely go
  wrong with the tool library?"
- *(user)*: "Someone gets hurt with a power tool and we get sued. Tools
  go missing and never come back."
- *(AI) builds*:
  - *(AI) A risk the user didn't name*: liability if a tool is returned
    damaged and the next borrower gets hurt using it unknowingly.
  - *(AI) A slower-burn risk*: volunteer burnout — a library run on
    volunteer labor quietly stops functioning within a year if no one
    owns it.
  - *(AI) A reputational risk*: uneven access — if popular tools are
    always checked out, the people who need them most give up first.
- Return the floor: "Anything else under black hat before we move to
  yellow — benefits?"

**Your build move** — Add material that stays strictly inside the
current hat — never a build that quietly imports another hat's
judgment, like sneaking a "but on the bright side" into a black-hat
round.

**Where it goes wrong** — The most common failure is hat-bleed: someone
under yellow hat (benefits) starts hedging with risk language, which
guts the point of isolating the perspectives. If you catch yourself or
the user doing this, name it out loud — "that's a black-hat thought,
let's park it for that round" — rather than letting it slide.

**Bail when** — A hat produces almost nothing (two rounds of thin
answers under, say, the green hat) — that's a sign the topic doesn't
have much for that particular lens, so move to the next hat rather than
forcing it, and consider stopping after four hats if five and six add
little.

## 7. Mind mapping

**What it is** — A central concept with branches radiating outward,
each branch expandable into its own sub-branches. The visual,
associative structure surfaces connections a linear list of ideas
hides, because related ideas end up spatially near each other.

**Best for** — Wide-open topics that need a structure to hold many
loosely related threads at once, or a user who thinks visually rather
than in ordered lists.

**Facilitate it** — Put the topic at the center and ask: "What are the
three or four main branches coming off the tool library — the big
categories it splits into?" The user proposes branches first. Then
expand together, branch by branch: pick one, ask what hangs off it,
follow it two or three levels before moving to the next branch.

**Example round**
- Prompt: "What are the three or four main branches off 'tool library'?"
- *(user)*: "Inventory, membership, location, and community events."
- *(AI) builds*:
  - *(AI) Expand a branch the user named*: under "location," sub-branch
    into fixed site vs. mobile trailer vs. locker network — three
    distinct paths worth tracking separately.
  - *(AI) Add a branch connecting two existing ones*: "skill-sharing" —
    sitting between "membership" and "community events," where members
    who are skilled with a tool teach others how to use it.
  - *(AI) Add a branch the user's map is missing entirely*: "funding" —
    grants, membership fees, and municipal partnership as three separate
    sub-branches worth mapping before this goes further.
- Return the floor: "Which branch do you want to go two levels deeper
  on next?"

**Your build move** — Add a branch that connects two existing branches
the user treated as separate — mind maps get their real value from the
connections between branches, not just from the branches themselves.

**Where it goes wrong** — The map turns into a flat list wearing a
different shape — every branch stays one level deep because the
facilitator moves on before expanding any of them. If that's happening,
stop adding new top-level branches and push one existing branch two
levels deeper before touching another.

**Bail when** — The map has sprawled past six or seven top-level
branches with none developed past one level; that's too broad to be
useful — pick the two or three branches with the most energy and either
develop those with Mind mapping or move to a narrower technique like
SCAMPER on just one branch.
