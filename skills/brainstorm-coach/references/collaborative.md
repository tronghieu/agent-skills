# Group 3 — Collaborative

Three techniques built around the back-and-forth itself — momentum
through alternating turns, breadth through parallel generation, or a
jolt through a random prompt. Reach for this group to warm a session up,
to recover energy mid-session, or when a topic benefits from breadth
before commitment. The examples below use one running generic topic — a
community tool-lending library (a place where neighbors borrow tools
instead of buying them) — so the same idea can be seen through several
techniques.

## 8. Yes-and building

**What it is** — Improv's core rule applied to ideation: the user offers
an idea, the AI extends it with a genuine "yes, and," the user extends
that back, alternating — no blocking, no "yes, but," just compounding
additions that keep the momentum moving forward.

**Best for** — Warming up a session, or any moment that needs pure
momentum over structure — the first few minutes, or a recovery after a
technique has gone flat.

**Facilitate it** — Ask for one starting idea, any size: "Give me one
idea for the tool library, any size." Then alternate turns explicitly —
user idea, AI "yes, and" on that exact idea, user "yes, and" on the AI's
addition — three or four exchanges before resetting with a fresh seed
idea rather than letting one thread run forever.

**Example round**
- Prompt: "Give me one idea for the tool library, any size."
- *(user)*: "A tool library that also has a repair bench where you can
  fix your own stuff."
- *(AI)*: "Yes, and — the repair bench has a volunteer 'fixer' there on
  weekends who'll help you if you're stuck, like a genius bar for
  broken lamps and wobbly chairs."
- *(user)*: "Yes, and the fixer could teach a short class each month —
  this month's is 'how to sharpen kitchen knives.'"
- *(AI)*: "Yes, and the classes get recorded and posted, so the
  knowledge doesn't leave when that one volunteer moves away."
- Return the floor: "Want to keep riffing on the repair-bench thread, or
  should I toss out a fresh seed idea to alternate on?"

**Your build move** — Your "and" must add something the idea didn't
already contain — a genuine extension, not a rephrasing of what the
user said with "yes, and" bolted on the front.

**Where it goes wrong** — The AI's "and" turns into a "but" wearing a
disguise ("yes, and — though that might be expensive") — any hedge
kills the momentum this technique exists to build. If a hedge is
genuinely important, hold it for the convergence phase, not here.

**Bail when** — A thread runs five or six exchanges without anything
genuinely new appearing (just elaboration on the same core idea); reset
with a fresh seed idea, or if that doesn't help either, move to a
technique with more built-in structure, like Brainwriting round-robin.

## 9. Brainwriting round-robin

**What it is** — An adaptation of silent brainwriting for a two-party
session: the user writes several ideas at once, without discussion, the
AI builds one variation on each independently, and the user then picks
which thread to develop further. Generating everything up front avoids
one early idea anchoring everything that follows.

**Best for** — Sessions where anchoring is a risk — one loud idea taking
over before other options get a fair hearing — or when the user wants
breadth before committing to any one direction.

**Facilitate it** — Ask for a burst, not one idea: "List three ideas for
the tool library, no explanation needed yet — just the ideas." Once the
three land, respond with one variation per idea — six threads total —
then hand the choice back: "which of these six do you want to push
further?"

**Example round**
- Prompt: "List three ideas for the tool library, no explanation yet."
- *(user)*: "1) A trailer that visits neighborhoods on rotation. 2) A
  membership that includes free classes. 3) A 'tool of the month'
  feature to get people trying things they'd never think to borrow."
- *(AI) builds — one variation per idea*:
  - *(AI) On the trailer*: instead of a fixed rotation, a text-in
    request system — the trailer goes where enough people ask for it in
    a given week.
  - *(AI) On the membership classes*: make one class a prerequisite for
    borrowing certain higher-risk tools (table saw, chainsaw), turning
    "free class" into "required onboarding" for those specific items.
  - *(AI) On tool of the month*: pair it with a show-and-tell night
    where people bring back what they made with it — turns a feature
    into a small recurring event.
- Return the floor: "Which of these six threads do you want to push
  further?"

**Your build move** — Give each of the user's three ideas its own
distinct variation rather than one strong build and two throwaways —
the technique's value is giving the user real options to choose among,
not showcasing your best idea.

**Where it goes wrong** — The AI's variations quietly become
replacements instead of variations — a build so different from the
user's original idea that it's really a new idea wearing the original's
name. Keep each build recognizably a version of the specific idea it's
attached to.

**Bail when** — The user's initial burst of three ideas is thin or
repetitive (variations on one theme); that's a sign the topic needs more
opening-up first — switch to What-if scenarios or Random stimulation to
generate more raw material before round-robin has anything to work with.

## 10. Random stimulation

**What it is** — Introducing a deliberately unrelated word, image, or
object and forcing a connection back to the topic. The randomness is
the point: a chosen prompt reflects associations that already exist in
someone's head, but a genuinely random one doesn't, so it reaches
somewhere a chosen prompt can't.

**Best for** — Stale topics that need a genuine jolt, or a session that
has started producing the same handful of ideas in different words.

**Facilitate it** — Supply a random word or object with no topical
connection — "umbrella," "the number seven," "a subway map" — and ask:
"What does [random word] suggest about the tool library — force the
connection, even if it feels like a stretch at first." Draw a genuinely
new random prompt each round; reusing one defeats the purpose, since the
value comes from the prompt being unpredictable.

**Example round**
- Prompt: "Random word: umbrella. What does that suggest about the tool
  library — force the connection, even if it's a stretch?"
- *(user)*: "An umbrella covers you when it's raining unexpectedly...
  maybe the library has an emergency-loan option — your pipe burst at
  10pm, is there any way to get a wrench right now?"
- *(AI) builds*:
  - *(AI) Push a second feature of umbrellas*: umbrellas are often
    shared or abandoned in stands at cafes — a no-checkout "community
    shelf" near the entrance for the ten most commonly needed basic
    tools, no formal borrowing process at all.
  - *(AI) Push "covers you when unexpected"*: a small kit of the five
    most common emergency tools, pre-bundled, that skips the normal
    membership process entirely for true emergencies.
  - *(AI) Push the shape of an umbrella*: something that folds down
    small and travels with you — a compact, portable version of the
    tool library that goes to a block party or work site instead of
    everyone coming to it.
- Return the floor: "Any of those worth developing, or want a new random
  word to try?"

**Your build move** — Push the connection one step past the obvious
first link the user found — if they connected "umbrella" to "shelter
from rain," reach for umbrellas' other features (they fold, they get
left behind, they're shared) rather than staying on the first link.

**Where it goes wrong** — The user reaches for the most generic possible
connection and stops there ("umbrella... protection... safety
features?") without forcing anything specific. If the connection feels
too easy, push back gently: "that's the safe link — what's a weirder
one hiding in there?"

**Bail when** — Two or three random words in a row produce only generic,
easy connections; that's often a sign the user is tired rather than the
technique failing — take an energy checkpoint before switching.
